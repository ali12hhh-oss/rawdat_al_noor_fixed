# روضة النور 🌙 — تطبيق تعليم القراءة والكتابة للأطفال

تطبيق Flutter تعليمي ثنائي اللغة (عربي/إنجليزي) لتعليم الأطفال الحروف والكلمات
عبر بطاقات تفاعلية، ألعاب تعليمية (مطابقة، ذاكرة، بناء كلمات، تتبع حروف، اختبارات)،
قصص مصوّرة، ونظام تقدّم ومكافآت.

## تشغيل المشروع

```bash
flutter pub get
flutter run
```

> ملاحظة: نفّذ الأمر التالي مرة واحدة إن أردت أن يقوم Flutter بتوليد/تحديث أي
> ملفات منصّة إضافية (خاصة مجلد `ios/` غير الموجود حالياً، أو دعم ويندوز/ماك/ويب):
> ```bash
> flutter create .
> ```

## ملخص الإصلاحات والإضافات التي تمت

### 🐞 أخطاء برمجية تم إصلاحها
- **`effects_manager.dart`**: استخدام خاطئ لـ `WidgetsBinding.instance.context` (غير موجود أصلاً) داخل
  `getFireworksEffect()` و `getHeartsEffect()` — تم تحويلهما لاستقبال `BuildContext` كمعامل.
- **`effects_manager.dart`**: تمرير `List<Widget>` مباشرة إلى `Positioned.fill(child:...)` بدل `Widget` واحد
  — تم لفّها بـ `Stack`.
- **`effects_manager.dart`**: عمليات حسابية `int` تم تمريرها لمعاملات `double` (`fontSize`, `moveY end`)
  — تم تصحيحها بإضافة `.toDouble()`.
- **`letters_screen.dart`**: استخدام خاصية `opacity` غير الموجودة في `TextStyle` — استُبدلت بـ `Opacity` widget.
- **`memory_game.dart`**: استخدام `AppTheme` دون استيراد `app_theme.dart` (خطأ ترجمة) — تم إضافة الاستيراد.
- **جميع ملفات `assets/json/*.json`**: كانت تحتوي BOM (UTF-8 Byte Order Mark) في بداية الملف، ما يجعل
  `json.decode()` في Dart يفشل بصمت (الكود يبتلع الخطأ ضمن `try/catch` ويُرجع قوائم فارغة) — أي أن
  **كل الحروف/الكلمات/القصص/المستويات كانت ستظهر فارغة تماماً في التطبيق**. تم إزالة الـ BOM من الملفات الخمسة.
- **`pubspec.yaml`**: استيراد `flutter_localizations` مستخدم في `main.dart` لكنه غير مُعرّف كاعتمادية — تمت إضافته.
- **`pubspec.yaml`**: مسارات الأصول لم تكن تشمل `assets/svg/letters/arabic/` و `assets/svg/letters/english/`
  (فلاتر لا يُدرج المجلدات الفرعية تلقائياً) — تم إصلاح القائمة.
- **مشروع أندرويد غير مكتمل** (لن يُبنى إطلاقاً بالحالة السابقة):
  - `MainActivity.kt` غير موجود — تم إنشاؤه.
  - مجلد `res/` بالكامل (الأيقونات، الثيمات، شاشة البداية) غير موجود — تم إنشاؤه كاملاً.
  - `android/gradle.properties` كان فارغاً (بدون تفعيل AndroidX) — تمت إضافة الإعدادات القياسية.

### 🔌 فجوات وظيفية تم ربطها (كانت شاشات/ملفات معزولة لا تُستدعى من أي مكان)
- **شاشة الألعاب (`games_screen.dart`)** كانت تعرض رسالة "قيد التطوير" فقط ولا تُشغّل أي لعبة فعلياً —
  تم ربطها بتحميل بيانات الحروف/الكلمات وتشغيل الألعاب الخمس الفعلية.
- **متجر المكافآت (`reward_store_screen.dart`)** لم يكن له أي مسار تنقّل إليه من التطبيق — تمت إضافة
  مسار `/rewards` وزر وصول من الشاشة الرئيسية، وتفعيل خصم النجوم الفعلي عند الشراء
  (أضيفت `spendStars()` في `ProgressProvider`).
- **شاشة تفاصيل القصة (`story_detail_screen.dart`)** لم تكن مربوطة بقائمة القصص — تم ربطها.
- **حفظ نتائج الألعاب**: الألعاب الخمس لم تكن تحفظ نتيجتها في `ProgressProvider` رغم وجود
  `gameScores` في نموذج التقدّم — تمت إضافة الاستدعاءات اللازمة.
- **الوضع الليلي**: كان هناك مفتاح تخزين (`keyDarkMode`) مُعرّفاً ولا يُستخدم أبداً — تم بناء
  `SettingsProvider` + `AppTheme.darkTheme` وربطهما بالكامل مع مفتاح تفعيل في شاشة الإعدادات.
- **`flutter_local_notifications`** كانت اعتمادية مُعرّفة في `pubspec.yaml` ولا تُستخدم — تم بناء
  `NotificationService` لاستخدامها فعلياً (إشعار تذكير بالتدرّب).

## 🔧 تحديث ثانٍ: تحديث Gradle/AGP/Kotlin إلى الإصدارات الحديثة (بعد فشل بناء GitHub Actions)

بعد تجربة أولى للبناء عبر GitHub Actions، تبيّن أن إعدادات Android في الحزمة الأصلية كانت **قديمة جداً**
ولن تُبنى على Flutter الحديث إطلاقاً. تم تحديث:

| الأداة | القديم | الجديد |
|---|---|---|
| Android Gradle Plugin (AGP) | 7.3.0 | **8.11.1** |
| Kotlin | 1.9.0 | **2.2.20** |
| Gradle Wrapper | 7.5 | **8.13** (الحد الأدنى المطلوب فعلياً لـ AGP 8.11 حسب توثيق Android الرسمي) |
| قالب `settings.gradle` | القديم (`apply from:` عبر `app_plugin_loader.gradle`) | القالب الحديث (`pluginManagement` + كتلة `plugins {}`) |
| `compileSdk` / `targetSdk` | غير محدد صراحة | **36** (مطلوب من `audioplayers_android` و `shared_preferences_android`) |
| JDK المطلوب للبناء | — | **17** (متطلب إلزامي لـ AGP 8.x) |
| حد Dart SDK الأدنى في `pubspec.yaml` | `>=3.0.0` | **`>=3.3.0`** |

كذلك تم اكتشاف وإصلاح مشكلة أخرى حرجة: **`android/gradlew` كان ملفاً فارغاً تماماً (٠ بايت)** وملف
`android/gradle/wrapper/gradle-wrapper.jar` **غير موجود إطلاقاً** — أي أن أي محاولة بناء (محلياً أو عبر CI)
كانت ستفشل فوراً بغض النظر عن إصدارات AGP/Kotlin. تم:
- إعادة كتابة `gradlew` و `gradlew.bat` بالسكربتات القياسية الصحيحة.
- إضافة خطوة في `flutter-build.yml` تُولّد `gradle-wrapper.jar` الأصلي تلقائياً عبر Gradle الرسمي
  (`gradle wrapper --gradle-version 8.13 --distribution-type all`) قبل خطوة البناء — لأن هذا الملف
  ثنائي (binary) يتطلب اتصال إنترنت لتوليده بشكل موثوق، وهو متوفر في بيئة GitHub Actions.

**للتطوير محلياً:** افتحوا المشروع في Android Studio أو نفّذوا مرة واحدة داخل مجلد `android/`
(بوجود Gradle مثبّت لديكم):
```bash
gradle wrapper --gradle-version 8.13 --distribution-type all
```
أو ببساطة دعوا Android Studio يُصلحه تلقائياً عند أول مزامنة (Gradle Sync).

## 📦 تحديث ثالث: تحديث مكتبات (حزم) Flutter القديمة

بعض الحزم في `pubspec.yaml` كانت مثبّتة بنطاقات (`^`) تمنع الترقية التلقائية عبر `flutter pub get`
لأسباب مختلفة (تثبيت على إصدار رئيسي قديم، أو نطاق `0.x` الذي يُعامله pub بصرامة أكبر). تم تحديث:

| الحزمة | القديم | الجديد | السبب |
|---|---|---|---|
| `flutter_tts` | `^3.8.3` | **`^4.2.5`** | الإصدار 4 يحتوي إصلاح صريح لتوافق Gradle/Kotlin مع AGP 8 (مذكور في changelog الحزمة رسمياً) |
| `flutter_local_notifications` | `^17.0.0` | **`^19.0.0`** | نطاق `^17.0.0` كان يمنع أي ترقية لأي إصدار 18/19/20/21 حتى مع `pub get` عادي |
| `flutter_launcher_icons` | `^0.13.1` | **`^0.14.4`** | نطاق `^0.13.1` (إصدار 0.x) يسمح فقط بترقيات الإصلاح ضمن `0.13.x`، وليس الانتقال لـ `0.14.x` |
| `flutter_lints` | `^3.0.0` | **`^5.0.0`** | تحديث دَوري لقواعد التحليل الساكن (لا يؤثر على البناء، فقط على تحذيرات `flutter analyze`) |

**بقية الحزم** (`provider`, `shared_preferences`, `audioplayers`, `flutter_svg`, `lottie`,
`flutter_animate`, `intl`, `connectivity_plus`, `permission_handler`, `url_launcher`, `hive`)
كانت نطاقاتها بصيغة `^X.Y.Z` مع `X >= 1`، وهذه الصيغة **تسمح تلقائياً** بأحدث إصدار فرعي متوافق
عند تشغيل `flutter pub get` (لا يوجد `pubspec.lock` في المشروع يُثبّتها على إصدار قديم)، لذا لم تكن
بحاجة لتعديل يدوي.

تم أيضاً التحقق يدوياً من أن كل استخدامات `flutter_local_notifications` و `flutter_tts` في
`notification_service.dart` و `tts_service.dart` / `audio_provider.dart` تستخدم فقط الدوال
المستقرة (`initialize`, `show`, `cancelAll`, `speak`, `setLanguage`...) التي **لم تتأثر** بأي من
التغييرات الجذرية (breaking changes) المذكورة في سجلّات التحديث لهذين الإصدارين — أي لا حاجة
لتعديل أي كود Dart بسبب ترقية هذه الحزم.

**ملاحظة**: `hive`/`hive_flutter` لم يعد المشروع نشط التطوير بقوة (آخر تحديثات رئيسية قديمة نسبياً)،
وهناك بديل مجتمعي أحدث باسم `hive_ce`/`hive_ce_flutter` بواجهة برمجية مطابقة تقريباً. لم نبدّله الآن
تجنباً لأي مخاطرة غير ضرورية، لكن يُستحسن التفكير بالانتقال إليه مستقبلاً إذا واجهتم مشاكل توافق.

أضفنا أيضاً ملف `analysis_options.yaml` (لم يكن موجوداً إطلاقاً) لتفعيل قواعد `flutter_lints` فعلياً
عند تشغيل `flutter analyze`.

## ⚠️ أشياء تحتاج انتباهكم قبل النشر

1. **مجلد `ios/` غير موجود إطلاقاً** في الحزمة الأصلية. لتشغيل التطبيق على آيفون، شغّلوا
   `flutter create .` في جذر المشروع (بوجود Flutter SDK مثبّت لديكم) لتوليده تلقائياً، ثم
   `flutter pub run flutter_launcher_icons` لتوليد أيقونة iOS أيضاً.
2. **الخطوط والأصوات الحالية بديلة مؤقتة** كما ذكرنا أعلاه — استبدالها بملفات احترافية موصى به بشدة.
3. **`applicationId`/`namespace`** في `android/app/build.gradle` لا يزال `com.example.rawdat_al_noor`
   — غيّروه إلى معرّف حزمتكم الفعلي قبل النشر على المتجر.
4. لم تتوفر لدينا بيئة تشغيل Flutter فعلية لتنفيذ `flutter analyze` أو بناء تجريبي؛ التدقيق تم يدوياً
   عبر قراءة كل ملفات الكود والتحقق من توازن الأقواس وصحة الاستيرادات ومطابقة الأصول. يُستحسن تشغيل:
   ```bash
   flutter analyze
   flutter pub get
   flutter run
   ```
   للتأكد النهائي بعد فتح المشروع لديكم.
